## üöÄ Key Features (VS Code Only)

- **Real-Time Analysis**: Blazingly fast security checks (<200ms) as you type.
- **10 Core Security Rules**:
    - **Hardcoded Secrets**: Detection of API keys and credentials.
    - **SQL Injection**: Prevention of unsafe query building.
    - **XSS Prevention**: Guarding against cross-site scripting.
    - **Command Injection**: Flagging dangerous system executions.
    - **Path Traversal**: Blocking unsafe file operations.
    - **Weak Crypto**: Warning about legacy algorithms (MD5, etc.).
    - **Dead Code**: Reducing attack surface by removing unused logic.
    - **Error Handling**: Ensuring safe exception patterns.
    - **Broken Auth**: Detecting weak session logic.
    - **Insecure Deserialization**: Preventing RCE via unsafe parsing.
- **100% Offline**: Powered by a local Go engine. No code ever leaves VS Code.
- **Quick Fixes**: One-click remediation for detected security risks.

## üèóÔ∏è Project Architecture

The project is divided into two primary sub-systems:

1.  **`engine/` (Go)**: The core security analysis engine. It handles AST (Abstract Syntax Tree) parsing and rule evaluation at high speeds.
2.  **`extension/` (TypeScript)**: The VS Code extension layer that provides the diagnostics UI, hovers, quick fixes, and the security dashboard.
